1. Три основные области работы с файлами в Git:
   - Рабочая папка: где вы работаете с файлами
   - Индекс (staging area): подготовленные файлы для коммита
   - Репозиторий: история коммитов

2. Команды:
   - git add: добавляет файлы в индекс
   - git commit: сохраняет изменения в репозитории
   - git push: отправляет изменения на удалённый репозиторий

3. Информативные сообщения помогают понять историю изменений.

4. git add имя_файла добавляет конкретный файл, git add . добавляет все изменения.

5. git status показывает текущее состояние рабочей папки и индекса.

6. git remote add origin URL связывает локальный репозиторий с удалённым. "origin" - стандартное имя для удалённого репозитория.

7. Флаг -u устанавливает upstream branch для автоматической синхронизации.

8. Ветка по умолчанию - main (ранее master). Важно для согласованности в команде.

9. git log показывает историю коммитов с автором, датой и сообщением.

10. Изменения не попадут в коммит.
